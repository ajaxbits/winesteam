# Fix: Ghost wine windows left on desktop after exit

## Problem

The cleanup function in `launch-steam.sh` doesn't kill `explorer.exe` or the parent `wine` processes that own macOS windows. When wineserver is killed first, these processes become orphaned but still registered with macOS's window server, leaving unresponsive ghost windows on the desktop.

## Changes

**File: `launch-steam.sh`** — line 146

Add `explorer.exe` and `wine` itself to the pkill pattern in the cleanup function:

```bash
pkill -9 -f "winedevice|steamwebhelper|steamservice|steam.exe|explorer.exe" 2>/dev/null
killall -9 wine 2>/dev/null
```

The `killall -9 wine` is needed because the `wine` binary processes don't match the Windows-path patterns — they show up as just `wine` in the process table and are the ones that own the macOS window server entries.

## Verification

1. Run `launch-steam.sh`
2. Ctrl+C to exit
3. Verify no ghost windows remain on desktop
4. Verify `osascript -e 'tell application "System Events" to get name of every process whose name is "wine"'` returns empty
